# 🚀 AI Bot - Инструкция по сборке релиза (v2.0)

## Архитектура сборки v2.0

**Новое в v2.0:** Используется **esbuild** для создания единого bundle вместо файл-за-файл обфускации. Это решает проблемы с `require()` внутри pkg snapshot.

## Установка зависимостей для сборки

```bash
npm install
```

Это установит все необходимые пакеты:
- `esbuild` — **сборка единого bundle** (НОВОЕ в v2.0)
- `pkg` — компиляция в .exe
- `fs-extra` — работа с файлами
- `glob` — поиск файлов
- `axios` — HTTP запросы для KeyAuth
- `javascript-obfuscator` — обфускация (опционально для v2)

## Настройка KeyAuth (ОБЯЗАТЕЛЬНО!)

### Процесс авторизации KeyAuth (2 шага):

1. **Init Session** — `GET /api/1.2/?type=init` → получаем `sessionid`
2. **License Check** — `POST /api/1.2/` с `sessionid` → проверка ключа

### Где настроить:

**Файл: `scripts/build-v2.js`** (в функции `generateEntryWithLicense()`):

```javascript
const KEYAUTH_CONFIG = {
  appName: 'Nullbit',              // ← Название приложения
  ownerId: '47IOqyDjNC',           // ← Owner ID из KeyAuth
  appSecret: 'daf5f53ecdce...',    // ← Secret ключ приложения
  version: '1.0'
};
```

> **Примечание:** Конфигурация встраивается прямо в .exe файл!

## Процесс сборки v2.0 (6 шагов)

1. **Очистка папки Release** — удаляет старые файлы
2. **Сборка bundle через esbuild** — **весь код в один файл**
   - Обрабатывает все `require()`
   - Встраивает все модули
   - Минификация для уменьшения размера
3. **Подготовка bundle** — копирование/минимальная обработка
4. **Создание entry point** — встраивание лицензии + кода бота
5. **Сборка .exe через pkg** — компиляция в бинарник
6. **Создание config.json и cleanup** — шаблон для пользователя

### Почему esbuild bundle лучше:

| Проблема v1.0 | Решение v2.0 |
|---------------|--------------|
| `require('./startBot.js')` не находился в snapshot | **Все модули в одном bundle** |
| Файл-за-файл обфускация сложна | **Один bundle → один .exe** |
| Обфускатор падал на 400MB+ | **esbuild minify достаточно** |
| Сложная структура зависимостей | **Всё встроено, нет внешних require** |

## Тестирование сборки

1. Закрой все копии `AIBot.exe` (иначе будет EBUSY ошибка)
2. Собери релиз: `npm run build`
3. Скопируй `Release/` на чистую машину (или VM без Node.js)
4. Заполни `config.json` валидным лицензионным ключом
5. Запусти `AIBot.exe`
6. Проверь:
   - ✅ Лицензия проходит (Session init → License check)
   - ✅ Бот стартует без `Cannot find module` ошибок
   - ✅ Бот подключается к серверу
   - ✅ PvP работает без лагов (50ms тикрейт сохранён)
   - ✅ Voice chat работает
